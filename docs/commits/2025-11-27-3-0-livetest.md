### 🚀 Release Candidate: Post-Live Test Stabilization

**Comprehensive fixes and hardening following live server testing.**

#### 🛠️ Infrastructure & Bootstrap
* **Fixed Setup UI:** Resolved duplicated buttons and `ConnectionState` crash in `SetupView` by using native decorators instead of listeners.
* **Channel Creation:** Fixed `TypeError` in `ensure_text_channel` regarding `overwrites`.
* **Wipe Command:** Added missing import `is_asdrubot_channel` to fix `NameError`.
* **Storage:** Added `infra` and `tzclocks` to `state.json` persistence to ensure role IDs and timezones survive restarts.

#### 🌍 Timezones
* **Validation Fix:** Fixed `TypeError` in `/tz edit` when the timezone parameter is optional/None.
* **Persistence:** Ensured timezone configurations are correctly saved/loaded via `storage.py`.

#### 🧪 Status & Effects
* **Loading Fix:** Added explicit import of `status.builtin` in `cogs/status.py` to ensure effects (Paralyzed, Silenced, etc.) are registered on startup.
* **Notification Routing:** Updated status notifications to use `send_to_player`, prioritizing private Role Channels over DMs.
* **Syntax Fix:** Resolved indentation error in `/effects heal` loop.

#### ⚔️ Actions & Logs
* **Note Persistence:** Refactored `enqueue_action` in `core/actions.py` to correctly merge the `payload` (notes) into the record.
* **Visual Overhaul:** Redesigned `/actions logs` to be cleaner (emojis, hidden IDs, bold targets).
* **Targeting:** Removed hardcoded `alive` check in `/act` to allow interaction with dead players (e.g., for Revivers/Mediums).

#### 🏗️ Core Refactor: Single Game Channel
* **Unified Channels:** Replaced `day_channel` and `night_channel` with a single `game_channel` concept across `state`, `infra`, `phases`, and `moderation`.
* **Commands:** Updated `/set_channels` and internal helpers to reflect the single-channel architecture.

#### 💀 Lifecycle (Kill/Revive)
* **Role Swapping:** Fixed `NameError` in `players.py` by importing `apply_alive_dead_role`.
* **Redundancy:** Simplified `kill` and `revive` commands to rely on the robust `set_alive` wrapper.
